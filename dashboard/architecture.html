Core architectural properties demonstrated
🔐
Ed25519 Two-Layer Signing
Registry signs AgentAddr. Agent signs AgentFacts. Canonical JSON serialization (sort_keys) prevents field-reorder attacks. Any modification invalidates the signature immediately.
--tamper flag injects false agent_id → caught
🚫
Sub-Second Revocation
POST /revoke/{id} flips a single DB bit. Client checks GET /status/{id} after every AgentAddr verification. Revoked agents are rejected before facts are even fetched.
DNS TTL: hours. NANDA revocation: <100ms
⚡
TTL-Based Caching
Client caches AgentAddr by agent name. Cache respects TTL field from AgentAddr. Second resolve within TTL: zero index calls. This is the mechanism behind 10,000× write load reduction.
cache-demo: 1st=22ms → 2nd=0ms index latency
🏗️
Two-Tier Decoupling
AgentFacts update independently of the index via PATCH /agents/{slug}/agent-facts. Re-signed in-place. Endpoint rotations, score updates, capability changes — no index write required.
Index stable · Facts layer live
🔒
Privacy-Preserving Path
Dual URL model: PrimaryFactsURL (public) and PrivateFactsURL (selective access). In production, PrivateFactsURL would be hosted on IPFS/CDN. Same signed document, restricted distribution.
--private flag switches resolution path
📐
JSON-LD / W3C VC Compatible
AgentFacts carry @context and type: ["VerifiableCredential", "AgentFacts"]. Ed25519 proof block. Schema-compatible with W3C VC v2 — proof.type: Ed25519Signature2020 is the only remaining gap.
Full W3C VC: scoped out (fragmented lib ecosystem)

Architecture tiers
🗂️
Tier 1 — Lean Index
Registry stores only AgentAddr — routing pointer + crypto hash. Minimal write surface. High read availability. Signs every response.
port :8000 · SQLite · Ed25519
📋
Tier 2 — AgentFacts
Full capability metadata. Agent-signed. Updated independently of the index. Supports primary + private URL paths. Live PATCH without index writes.
port :8001 · JSON-LD · VC-compatible
🏢
Tier 2b — Enterprise Registry
Optional second hop for enterprise-routed agents. Independent signing key. Enterprise controls access, visibility, and key rotation without touching NANDA.
port :8002 · independent keypair
